# 6.1 科研数据采集

本目录对应教材第 6.1 节，介绍通过 CNKI、Web of Science 和 Scopus 的官方检索、筛选与导出功能准备科研文献数据。

本节不提供 API 调用程序，不编写网络爬虫，不模拟登录，也不采集需要授权的数据库内容。仓库中不包含从科研数据库实际批量导出的受版权限制数据。

## 官方检索与导出流程

不同数据库的页面名称和可选格式可能不同，但基本过程一致：

1. 明确研究主题、文献语言、时间范围和所需字段。
2. 通过学校图书馆、科研机构或数据库官方网站的正规入口访问平台。
3. 在主题、题名、作者、机构或 DOI 等官方检索字段中建立检索式。
4. 按年份、文献类型、学科、语言和来源等条件筛选结果。
5. 抽查题名与摘要，判断检索结果是否符合研究问题。
6. 勾选授权范围内的记录，通过平台的“导出”或参考文献管理功能保存数据。
7. 打开导出文件，核对记录数、关键字段、字符编码和重复记录。
8. 保存检索式、筛选条件、导出日期、记录数和文件名，以保证过程可追溯。

CNKI 常用于中文期刊、学位论文、会议论文等文献检索；可按主题、篇名、作者或机构检索，再使用平台提供的导出或参考文献管理功能。

Web of Science 可在选定的数据库范围内按主题、题名、作者、机构或 DOI 检索，并按年份、文献类型、研究方向和语言筛选。常见导出字段包括题名、作者、作者单位、来源、摘要、关键词、年份、DOI 和被引次数。

Scopus 的流程同样包括检索、筛选、选择记录、选择元数据字段和导出。一般统计可选择 Excel/CSV；导入 EndNote、Zotero 等文献管理软件时，可在平台支持范围内选择 RIS、BibTeX 等交换格式。

## 模板说明

`templates/research_export_template.csv` 用于统一不同数据库的文献导出字段：

```text
title,authors,affiliations,year,source,document_type,abstract,keywords,doi,citation_count,database
```

文件中的两行数据均为虚构教学示例，不对应真实数据库记录。

`templates/collection_record_template.csv` 用于记录一次文献数据采集过程：

```text
database,search_topic,search_date,search_field,time_range,document_type,export_fields,export_format,record_count,file_name
```

建议每次导出后追加一行，完整保留数据库、检索主题、日期、字段、范围、格式、记录数和文件名。

## 数据整理建议

- 保留一份未经修改的原始导出文件，复制后再整理。
- 统一不同数据库的字段名和日期格式。
- 优先使用 DOI 去重；缺少 DOI 时，可结合题名、作者和年份判断。
- 检查摘要、关键词和机构字段是否被完整导出。
- 数据库持续更新，同一检索式在不同日期的结果数可能不同，因此必须记录检索日期。

## 授权与版权

用户应遵守科研数据库的使用规则、机构授权范围、导出数量限制和数据版权要求。不要共享机构账号，不要绕过访问限制，不要把受许可约束的批量记录公开上传到 GitHub。需要公开研究数据时，应确认数据库许可和所在机构的管理要求，只发布允许公开的字段、统计结果或经过合法处理的数据。
